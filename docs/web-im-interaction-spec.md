## 0. 一句话产品定义

> 一个**给运营负责人和评委用**的 Web IM。把 4 个 AI Agent(运营经理 + KOL增长 / SEO内容 / 社媒分发)的对话、协作、审批和运行状态,**统一在浏览器里完成**。

---

## 1. 角色与心智模型

| 角色 | 心智 | 关心什么 |
|---|---|---|
| 运营负责人(主用) | "把任务交给团队,不用每条都盯" | 任务进度、审批节点、Bot 之间的对话是否合理 |
| 评委 / 演示观众 | "想看 AI 真的在干活,而不是 PPT" | 多 Agent 协作的过程感、可视化、卡片化输出 |
| Agent(被使用对象) | "我是团队成员,不是工具" | 头像、名字、状态、对话边界 |

**核心心智**:这是 **"团队 IM"**,不是"AI 单点对话框"。它的形式语言更像 Slack / 飞书,而不是 ChatGPT / Claude.ai。

```
│  · Agent 3   │  └────────────────────────┘    │                  │

```

### 2.1 三栏的责任边界
- **Sidebar(导航)**:让用户切换"我现在在和谁说话"。
- **ChatArea(对话)**:消息收发的主战场,包括 Bot 之间的协作。
- **DetailsPanel(上下文)**:不参与对话本身,只解释"现在到底在发生什么"。可折叠为 0 或图标条。

## 3. 关键组件清单

> 每个组件至少给 4 个状态:default / hover / active / disabled。+ 必要时的 loading / error。

### 3.1 TopBar
- 元素:Logo · 标题文字 · 版本/环境徽章 · 在线 Agent 计数 · 设置入口 · (扩展)用户菜单
- 高度 48px;贯穿全宽;不滚动

### 3.2 Sidebar · 搜索框
- placeholder「搜索会话或 Agent」
- 焦点态高亮边框
- 键盘 `⌘K` / `Ctrl+K` 唤起(本期可不实现,设计预留)

### 3.3 Sidebar · 会话项(`ConversationItem`)
关键状态:
| 状态 | 视觉提示 |
|---|---|
| default | 普通底色 |
| hover | 浅高亮底 |
| active(当前会话) | 强调底 + 强调边框 |
| 有未读 | 右侧红点+数字徽章 |
| Agent 私聊 | 头像右下角带状态点(在线/忙碌/离线/重启中) |
| 群聊 | 头像位用 `#` 占位或多人头像堆叠 |

每行字段:
- 第一行:**会话名**(粗) ·(右)最近消息时间(11px)
- 第二行:**最近消息片段**(单行省略) · (右)未读数

### 3.4 Sidebar · 用户卡(底部)
- 头像 + 用户名 + 角色标签(如"运营负责人")
- 点击未来挂用户菜单

### 3.5 ChatArea · Header
- 会话标题 + 成员描述(如「5 成员 · 运营经理 · KOL增长 · SEO内容 · 社媒分发」)
- 群聊时,右侧显示成员头像堆叠(最多 4 个 + "+N")
- 每个头像右下角带状态点

### 3.6 ChatArea · 消息气泡(4 种 kind)

**总规则**:
- 用户消息靠右、Bot/系统消息靠左。
- 每条消息上方一行:作者名 + 时间;头像在外侧(用户右、Bot 左)。
- 鼠标悬停整行,悬浮操作按钮(复制/引用/重新生成)出现在一侧。

#### 3.6.1 文本气泡 `kind=text`
- 支持 Markdown(标题、列表、代码块、表格)
- 流式态:末尾有"光标"指示符,持续脉动
- **mention 高亮**:`@运营经理` 渲染为强调色块,点击可跳转到该 Agent 私聊

#### 3.6.2 工具调用卡 `kind=tool-call`
> 表示 Bot 调了一次工具(MCP / 内置)
- 折叠态(默认):一行,显示工具图标 + `tool.name` + 耗时 + 展开箭头
- 展开态:显示 `args`(JSON,等宽字体)+ `result`(文本/JSON)
- 状态点:成功(绿)/进行中(黄,脉动)/失败(红)
- **设计意图**:让用户*看到 AI 在干活*,但不喧宾夺主。默认不打扰、点开有细节。

#### 3.6.3 审批卡 `kind=approval`(★ 最重要)
> Bot 在外发动作前必须发起审批,这是产品的差异化亮点。
- 标题区:警告图标 + 标题 + 风险标(low/medium/high) + 简介
- 文案预览区:显示即将发出的内容(可截断)
- 目标列表区:展示触达对象(标签云),超过 5 条折叠
- 按钮区:**通过**(主操作,绿色调) · **驳回**(次操作,带备注输入弹层)
- 倒计时提示:5 分钟自动驳回(灰色脚注)
- 状态:pending / approved(显示通过人和时间)/ rejected(显示备注)/ expired
- **设计意图**:让用户感觉"自己仍然掌控"。视觉上必须比普通气泡更"重",拉走注意力。

#### 3.6.4 任务卡 `kind=task`
- 单行核心字段:`#task-id` · 负责人 · 状态徽章
- 标题
- 进度条(可选)
- 点击可跳转到任务详情(本期可只悬浮 tooltip)

### 3.7 ChatArea · Composer(输入器)
- 多行 textarea(默认 2 行,自动撑高至 8 行)
- 工具栏(textarea 下方一行):
  - 附件按钮(本期 P2,可灰显)
  - `@` 按钮 → 弹 Agent 选择器
  - 提示模板按钮(图标可用 Sparkles)
  - **发送按钮**(主操作,有/无内容两态)
- 底部一行 11px 提示:`Enter 发送 · Shift+Enter 换行 · @ 触发 Agent`
- **聚焦态**:整个 Composer 容器边框转为强调色

### 3.8 ChatArea · @ Mention 选择器(弹出)
- 用户输入 `@` 触发
- 弹出列表:仅当前会话成员中的 Agent
- 每行:头像 + 名字 + 状态 + 简短职责描述
- 键盘 ↑↓ 选,Enter 确认

### 3.9 DetailsPanel · 三个段(可折叠)

#### 3.9.1 Agent 状态
- 卡片化,每行一个 Agent
- 头像 + 状态点 · 名字 · 职责 · pid
- hover 显示"重启""查看日志"等(本期可只占位)

#### 3.9.2 当前任务
- 卡片列表:任务编号 · 负责人 · 状态徽章 · 标题
- 状态徽章 5 种颜色(queued / running / blocked / done / failed)
- 点击进入任务详情视图(本期可不实现,留入口)

#### 3.9.3 工具调用日志
- 时间序列单行:状态点 · 时间 · 工具名
- 点击展开同 3.6.2 的卡片
- 默认折叠

### 3.10 空状态
- ChatArea 无消息时:居中文字「还没有消息 / 用 @ 召唤一个 Agent 开始任务」
- Sidebar 无会话:CTA「+ 新建群聊」

### 3.11 加载与错误
- 流式态:Bot 气泡末尾光标 + 头部"正在输入…"
- 网络断开:顶部细横条「连接已断开,正在重连… ↻」
- 消息发送失败:气泡角标红色感叹号 + hover 显示重试

## 4. 关键流程时序

### 4.1 私聊一来一回
```
[用户] 输入 → Enter
       ↓ 用户气泡立刻出现(乐观渲染)
[系统] 头部状态条 "运营经理 正在输入…"
       ↓
[Bot]  气泡占位出现,内容随流式 chunk 增长(打字机)
       ↓ 流完,光标消失,头部状态条消失
```

### 4.2 群聊 @ 转派(★)
```
[用户] 在战情室输入 "@运营经理 本周锁定 30 个 KOL"
       ↓ 高亮 mention,Enter
[运营经理] 流式回:"收到。@KOL增长 拉一个候选池…"
       ↓ 系统识别新的 mention
[KOL增长] 自动唤醒(无需用户介入),开始流式回复
       ↓
[KOL增长] 调工具 → 出工具调用卡 → 出文本气泡 → 出审批卡
```
**设计要点**:Bot 之间转派要有**视觉连贯感**——例如同一群聊连续多条消息,头像与作者名的节奏要让人**一眼看出"这是一场 4 人讨论"**。

### 4.3 审批
```
[Bot]  推送审批卡(pending,带 5min 倒计时)
[用户] 点"通过" → 卡片折叠为成功态(显示"已通过 · {你} · 16:02")
                ↓ Bot 继续后续动作
[用户] 点"驳回" → 弹层填备注 → 提交 → 卡片折叠为驳回态(显示备注)
                ↓ 备注作为 user 消息追加 → Bot 修改后再发起
[超时] 卡片折叠为 expired 态(灰色),提示"已自动驳回,请人工再次触发"
```

### 4.4 演示模式(`?demo=kol-flow`)
```
[进入] 自动选中"演示群"
[页面顶部出现引导条] "KOL 触达完整流程演示" + [开始] 按钮
[点击开始] 系统按预设剧本,逐条注入 user 消息,驱动 4 个 Agent 协作
           ↓
[右侧时间轴] 高亮当前阶段(候选池→邮箱抓取→DM 起草→审批→发送→跟进)
```

## 5. 视觉与无障碍最低要求

> 这是给 OpenDesign 的**底线约束**,不是设计稿。具体设计可以远离这些占位值。

| 维度 | 要求 |
|---|---|
| 主题 | 默认深色;**预留**浅色(变量化即可) |
| 对比度 | 正文文本 ≥ 4.5:1;次要文本 ≥ 3:1 |
| 字号 | 最小 11px(辅助元数据)、12px(次要)、13-14px(正文)、16px+(标题) |
| 焦点态 | 所有可交互元素必须有可见焦点环(Tab 键) |
| 动效 | 流式打字机、状态点脉动、卡片展开使用 ease-out;**总动效预算** ≤ 200ms,避免眩晕 |
| 头像着色 | Agent 用语义色板,人类用户用中性色 |
| 状态点配色 | online=绿 · busy=黄(脉动) · offline=灰 · restarting=红(脉动) |

## 6. 与其他工具的视觉差异化(请避免雷同)

- **不要做成 ChatGPT 单列样式**:它是单 Agent 工具,我们是团队。
- **不要做成飞书完全镜像**:可以借鉴会话栏,但消息内卡片(审批/工具调用)是我们独有的。
- **不要 Slack 蓝紫风**:容易撞色,可以走更"操作台/雷达室"的科技感,或更克制的"工作室"质感。
- **建议方向**(供参考,不限制):
  - A. 雷达室科技感:暗底 + 强对比、卡片有微弱发光、Agent 像舰桥成员
  - B. 工作室静默感:接近 Linear / Raycast,信息密集但呼吸感强
  - C. 编辑部气质:把 SEO/KOL/分发 三条线渲染得像编辑部门牌
